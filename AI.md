공통 개발 지시문

너는 Python 기반 NLP/검색 알고리즘 개발자다.

현재 프로젝트는 국회 법률발의안의 제안이유 및 주요내용 텍스트를 기반으로 법안 간 유사도를 계산하는 실험 프로젝트다.

데이터는 test_dataset/full_dataset.json에 있으며 구조는 대략 다음과 같다.

{
  "bills": [
    {
      "bill_id": "...",
      "bill_name": "...",
      "summary": "제안이유 및 주요내용\\n\\n ...",
      "categories": [...]
    }
  ]
}

주의할 점:

summary는 단순 요약문이 아니라 국회가 제공하는 제안이유 및 주요내용 원문이다.
summary 안에는 보통 다음 요소가 섞여 있다.
현행법 설명
문제점 또는 개정 필요성
개정 내용
개정 목적
개정 조문
단순히 summary 전체를 하나의 임베딩으로 만들면 유사도 정확도가 떨어질 수 있다.
최종 목표는 문장 표현 유사도가 아니라 실제 법률의 의미, 적용 대상, 규제 대상, 법적 효과의 유사성을 판단하는 것이다.
단계별로 구현하며, 각 단계마다 결과 파일을 저장하고 이전 단계와 비교 가능해야 한다.
기존 코드는 최대한 보존하되, 기능별 파일로 분리하여 실험하기 쉽게 만들어라.
모든 출력 JSON은 ensure_ascii=False, UTF-8 인코딩으로 저장한다.
Windows 환경에서도 한글 출력이 깨지지 않게 처리한다.
대규모 데이터에서 O(n²) 전체 결과 저장은 위험하므로, 가능한 경우 각 법안별 top-k 결과만 저장한다.
각 단계마다 실행 방법과 결과 파일명을 주석 또는 README 형식으로 남겨라.
중요: dataset에 categories 필드가 존재할 수 있지만, 이는 실제 서비스에서 주어지는 값이 아니라 테스트를 위해 사람이 직접 붙인 평가용 라벨이다. 따라서 categories는 임베딩 입력, 유사도 계산, 후보군 검색, 법률 의미 프레임 생성, 하이브리드 점수 계산에 절대 사용하지 않는다. categories는 평가 분석이나 사람이 결과를 검토할 때 참고용으로만 사용할 수 있다.
