| 메소드                        | 입력 방식                                               | 핵심 아이디어                               | 장점                                      | 단점                               |
| -------------------------- | --------------------------------------------------- | ------------------------------------- | --------------------------------------- | -------------------------------- |
| `raw`                      | `bill_name + summary` 전체                            | 법안 제목과 제안이유 및 주요내용 전체를 그대로 SBERT 임베딩  | 가장 단순한 baseline. 구현 쉬움                  | 현행법 설명, 형식 문구, 반복 표현까지 모두 섞임     |
| `structured`               | 현행법 / 문제점 / 개정내용 / 조문으로 나눈 뒤 라벨 붙임                  | summary 구조를 명시해서 SBERT가 문단 역할을 구분하게 함 | 순위 정렬이 잘 될 수 있음. nDCG, MRR에 강하게 나올 수 있음 | 분리 규칙이 틀리면 오히려 노이즈 생김            |
| `problem_proposal`         | 문제점 + 개정내용만 사용                                      | 현행법 설명을 빼고 “왜 바꾸는지, 어떻게 바꾸는지”만 비교     | 법안의 실제 개정 취지를 잘 잡음                      | 조문, 배경, 법률 체계 정보는 약해짐            |
| `weighted_field`           | full/current/problem/proposal/article 각각 임베딩 후 가중합  | 필드별 중요도를 다르게 반영                       | 여러 정보를 종합함. top-10 후보 품질이 안정적일 수 있음     | 가중치가 잘못되면 잡음도 같이 커짐              |
| `cleaned_problem_proposal` | problem + proposal에서 형식 표현 제거 후 SBERT               | “현행법은”, “그런데”, “이에” 같은 상투어 제거         | 불필요한 문체 노이즈 감소                          | 너무 많이 지우면 문맥이 깨질 수 있음            |
| `keyword_tfidf`            | problem + proposal을 키워드화 후 TF-IDF                   | 의미 임베딩이 아니라 핵심 단어 겹침으로 비교             | 같은 법률 용어·핵심 키워드가 겹칠 때 강함                | 단독 성능은 약할 수 있음. 문맥·법적 효과 구분이 어려움 |
| `hybrid_cleaned`           | cleaned SBERT + keyword TF-IDF + article similarity | 의미 유사도, 키워드 유사도, 조문 유사도를 결합           | 상위 추천 정확도 개선 가능. P@5/P@10에 강할 수 있음      | 결합 가중치 튜닝 필요. 순위 정렬은 불안정할 수 있음   |

| 지표                    | 의미                   | 계산 기준                     | 높으면 좋은 뜻                        |
| --------------------- | -------------------- | ------------------------- | ------------------------------- |
| `P@5`                 | Precision@5          | 상위 5개 추천 중 관련도 3점 이상인 비율  | 사용자가 상위 5개만 봤을 때 쓸 만한 추천이 많음    |
| `P@10`                | Precision@10         | 상위 10개 추천 중 관련도 3점 이상인 비율 | top-10 후보군 안에 관련 법안이 많이 포함됨     |
| `nDCG@10`             | 순위 품질 지표             | 관련도 높은 법안이 상위에 있을수록 높은 점수 | 단순히 찾는 것뿐 아니라 “좋은 법안을 위에 잘 배치함” |
| `MRR`                 | Mean Reciprocal Rank | 첫 번째 관련 법안이 몇 위에 나오는지 평가  | 사용자가 첫 관련 결과를 빨리 발견함            |
| `AvgRel`              | Average Relevance    | 추천 결과들의 평균 관련도 점수         | 추천된 후보 전체의 평균 품질이 좋음            |
| `Legal Meaning Score` | 법률 의미 유사도 점수         | 이슈/대상/효과/범위/조문 세부 점수 가중합  | 단순 문장 유사도가 아니라 실제 법률 의미가 유사함    |
